问题合辑

1.nmp和node不能全局应用

  • 第一种安装方法：

卸载原有的手动安装的所有:sudo rm -rf /usr/local/node-v21.5.0-linux-x64/

重新安装sudo apt update,sudo apt install nodejs npm

有问题就sudo apt install -f,sudo apt autoremove,sudo apt install nodejs npm

  • 第二种安装方法：

    进入官网https://nodejs.org/en下载最新版

    sudo tar -xvf xxxxxxxxx.tar.xz

    cd xxxxxxxxx

    sudo cp -R * /usr/bin

    建立软连接(→)

    sudo ln -s /usr/local/bin/npm /usr/bin/npm/

    sudo ln-s /usr/local/bin /node /usr/bin/node/

    确保usr/local/bin/在环境变量中

    echo $PATH

    export PATH=$PATH:usr/local/bin/

2.后端启动报错：

  • error obtaining VCS status: exit status 128

    Use -buildvcs=false to disable VCS stamping.

    这次的问题是仓库有一些修改尚未提交，提交到与远程仓库就好了

3.为什么我的git命令前都需要加sudo

仓库的所有人是root

  • sudo chown -R 1000:1000 仓库目录

4.推送git push时远程仓库拒绝连接

  • git remote set-url origin ssh连接

    如果ssh连接没有建立好需要：确定代理正在运行并且包含我的秘钥

    eval “$(ssh-agent -s)”

ssh-add -l

这时你也有可能没有生成秘钥，那就需要执行生成秘钥

ssh-keygen -t ed25519 -C “2789458XXX@qq.com“

ssh-add ~/.ssh/id_ed25519

将生成的公钥(~/.ssh/id_ed25519.pub)添加到github

运行以下命令将 SSH Agent 配置添加到 Git：

git config –global core.sshCommand “ssh -v -i ~/.ssh/id_ed25519 -o ‘StrictHostKeyChecking=no’ -o ‘UserKnownHostsFile=/dev/null’”

“不能锁定配置文件 /home/zsy/.gitconfig: 权限不够” 的错误表明你的用户没有足够的权限修改全局的 Git 配置文件

手动编辑~/.gitconfig添加

[core]

sshCommand = ssh -v -i ~/.ssh/id_ed25519 -o 'StrictHostKeyChecking=no' -o 'UserKnownHostsFile=/dev/null'

5.在github上有两个差异过大的分支想要合并

  • 只能手动git

    创建一个空的本地仓库 git init

    git clone xxxx.git

    cd 本地仓库

    git merge origin master —allow-unrelated-hestories

    git add .

    git merge —continue

    git commit -m “”

    git push origin main

    git push origin —delete master
