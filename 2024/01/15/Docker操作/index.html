1.简介

  • 广义上：

    虚拟化的技术。下载完就使用。将环境变量配置好了。

  • 优势在于：

更高效的利用系统资源；容器不需要硬件虚拟，运行系统的额外开销

更快启动时间；

一致的运行环境；

持续的交付和部署。

更轻松的迁移。

更轻松的维护和扩展。（分层存储、镜像）

  • 实际上：

是容器

  • 与传统虚拟机比较：

  • 镜像与容器：

    用户基于镜像来运行容器。镜像是Docker生命周期的“构建”、“打包”部分。

    理解：镜像是类,容器是对象

    容器放置服务基于镜像启动，运行一个多个进程。容器像是Docker生命周期的“启动”“执行”

2.常用命令

  • 镜像：

  • 容器

守护方式创建容器后，进入容器命令行的指令：

docker exec -it myubuntu1 /bin/bash

守护方式运行容器，即使退出，容器仍然运行。

docker stop +id/name停止。

docker start +id/name开始。

删除镜像：rmi

删除容器：rm

  • mysql的部署：
    • 先拉取5.7版本的mysql镜像docker pull mysql:5.7
    • docker run -di - -name=xxx -p 3306:3306 -e MYSQL_ROOT_PASSWORD=root mysql:5.7(顺便配置了一波密码

  • nginx的部署：
    • 拉取docker pull nginx
    • 映射：docker run -di - -name=nginx -p80:80 nginx
    • 拷贝：docker cp /etc/nginx:/usr/local/mytry—>改名mv nginx conf
    • 挂载：docker run -di - -name=nginx -p 80:80 -v /usr/local/mytry/conf/:/etc/nginx nginx
  • redis的部署：
  • 拉取
  • 运行容器 同上不过p:6379
