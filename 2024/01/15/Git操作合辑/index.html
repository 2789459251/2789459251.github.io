4.时光穿梭

修改文件与文件提交

  • git add 文件
  • git commit -m “说明”
  • git reset HEAD:暂存区文件撤销操作
  • git log -5 —pretty=oneline log的简化输出指令

版本回退

  • git reset - -hard HEAD^ 退回上个版本（回退几个版本就加几个^）

  • cat s：看工作目录的文件内容。

  • git reset - - hard HEAD~[数字]：回退[数字]个版本。

  • 到未来版本：

  • git reset –hard 83fad5ea8ff31208c4cef8909e809f8e039d3623

问题：到第三个版本后

  • git log -5 —pretty=oneline看不到版本4、5的id了
  • git reflog //可以看HEAD历史的指向变化
  • 再使用
  • git reset - -hard id

文件删除

在工作区删除不影响git仓库

误删恢复：git checkout - - 文件名

从本地仓库拷贝到工作区

从本地仓库和工作区删除：git rm 文件名

但还是可以通过版本回退来恢复。
